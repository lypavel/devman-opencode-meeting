# Как вносить изменения

Проект — простая HTML-страница с JS и CSS, без сборщиков и бэкенда.
Разработка ведётся через ИИ-агента с помощью [opencode](https://opencode.ai).

## Установка opencode

Установите [npm](https://nodejs.org/en/download).

После установите opencode в соответствии с [его документацией](https://opencode.ai/docs/en#install).

После установки запустить в корне проекта:

```bash
opencode
```

Запуск игры: `./run.sh` (или указать порт: `./run.sh 3000`).

## Правила работы

- Не ломать существующую логику игры
- Соблюдать [styleguide](STYLEGUIDE.md)
- После изменений проверить все тест-кейсы и corner cases (см. ниже)
- Не добавлять внешних зависимостей
- Новый код писать через opencode — задачу описывают на русском, агент вносит изменения

## Тест-кейсы

- Таймер стартует с 00:00 при новой игре
- Таймер увеличивается раз в секунду
- Таймер останавливается при победе
- «Новая игра» сбрасывает таймер на 00:00 и запускает заново
- «Новая игра» сбрасывает счётчик ходов и прогресс по парам
- Ход увеличивается ровно на 1 после попытки открыть пару (после 2-й карточки)
- Нельзя открыть третью карточку, пока пара в проверке
- Совпавшая пара остаётся открытой и больше не кликается
- Победа появляется только после того, как найдены все пары

## Corner cases

### Блокировка доски (lockBoard)
- Клик по 3-й карточке, пока 2 открыты и не обработаны — не должен открываться
- Двойной клик по одной и той же карточке — не должен считаться второй попыткой
- Быстрая серия кликов во время таймаута (800 мс) — не должна ломать состояние

### Открытие/закрытие карточек
- Клик на уже открытую карточку (не matched) — игнорируется
- Клик на найденную пару (matched) — игнорируется
- Клик на matched-карточку сразу после находки (до снятия lockBoard) — игнорируется

### Совпадение и победа
- Нахождение последней пары — корректный подсчёт ходов, показ сообщения
- Начало новой игры после победы — сброс счётчика, сообщения, перемешивание

### Реинициализация
- Нажатие «Новая игра» во время таймаута 800 мс — lockBoard должен сброситься
- Нажатие «Новая игра» после победы — полный сброс
- Нажатие «Новая игра» с 1 открытой карточкой — сброс без ошибок

### Таймер
- Двойной вызов resetGame — старый интервал убит, timerInterval не дублируется
- Победа останавливает таймер — таймер не тикает после показа сообщения

### Массив flippedCards
- Очистка после совпадения и после несовпадения
- После resetGame массив пуст, даже если был прерван таймаут

### Счётчик ходов
- Ход засчитывается только когда открыты 2 карточки — не при каждом клике
- Сброс на 0 при новой игре
